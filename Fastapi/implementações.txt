2 - semana Atualização Nicolas:
Organizei o projeto – em pastas separadas para ficar profissional e fácil de manter:

core/ → cuida da conexão com o banco

models/ → define como os dados estão armazenados no banco

schemas/ → define como os dados serão enviados para o frontend

routers/ → define as rotas/endpoints que o frontend pode acessar

gibs.py → é o ponto de partida da API.

Conectei ao banco PostgreSQL na nuvem – onde o Gustavo vai salvar os resultados do modelo (lista de hotspots).

Criei a rota /hotspots – é o “caminho” que Ester com frontend vai acessar para receber a lista de coordenadas dos hotspots em JSON.

No final testei a API – ela está rodando, mostrando a estrutura dos dados no Swagger (/docs), e pronta para devolver os dados reais assim que houver registros no banco.
